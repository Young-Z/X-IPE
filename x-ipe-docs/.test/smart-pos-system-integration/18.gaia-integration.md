# [FSN/WFJ/FBP] SMART POS <-> GAIA Integration Design

> Integration design for SMART POS and GAIA for all three divisions, syncing TX data and sales target data for reconciliation and commission reporting.

## Integration Overview

GAIA receives sales transaction data and sales target data from SMART POS for reconciliation and commission calculations.

### Key Facts
- **Divisions:** FSN, WFJ, FBP (all three)
- **Integration Type:** File
- **Gateway:** Apollo CN

## Business Flow Impact

| Domain | Function | Downstream |
|--------|----------|------------|
| Sales Management | Sync TX Data To GAIA | Reconciliation with realtime TX data |
| Sales Management | Sync SaleTarget To GAIA | Commission with SaleTarget data |

## Integration Table View

| Division | Flow | Type | Source | Target |
|----------|------|------|--------|--------|
| FBP | Sync TX Data | File | SMART POS | GAIA |
| FSN/WFJ | Sync TX Data | File | SMART POS | GAIA |
| FBP | Sync SaleTarget Data | File | SMART POS | GAIA |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | Parallel run: TTPOS sends TTPOS TX, SMART POS sends SMART POS TX |
| Integration Gateway | Apollo CN required |
| Security | Defined at flow level |
