# [3 Divisions] SMART POS <-> AX TX Tender Reconciliation Integration Design

> Integration design for SMART POS and AX for TX tender reconciliation across all three divisions, syncing transaction data for automated financial reconciliation.

## Integration Overview

SMART POS sends transaction (TX) tender data to AX for automated reconciliation. Only SMART POS-generated data is sent (TTPOS migrated TX data is excluded).

### Key Facts
- **Divisions:** All three (FSN, WFJ, FBP)
- **Domain:** Inventory Management
- **Function:** Sync TX data for automated reconciliation
- **Gateway:** Apollo CN + Apollo APAC (distribution pattern)

## Business Flow Impact

| Domain | Function | Downstream |
|--------|----------|------------|
| Inventory Management | Sync TX data to automate Reconciliation | Automate |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | SMART POS only sends SMART POS-generated data (excludes TTPOS migrated TX data) |
| Integration Gateway | Apollo CN required; Apollo APAC required in distribution pattern |
