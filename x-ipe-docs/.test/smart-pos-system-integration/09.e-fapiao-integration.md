# [FBP/FSN] SMART POS <-> E-FAPIAO Integration Design

> Integration design for SMART POS and E-FAPIAO (electronic invoice system) for FBP and FSN divisions, handling electronic invoice issuance triggered by sales transactions.

## Integration Overview

E-FAPIAO is China's electronic invoice system. SMART POS provides order data; E-FAPIAO controls all invoice issuance logic. The flow is triggered when customers scan a QR code to request an invoice.

### Key Facts
- **Divisions:** FBP, FSN
- **Integration Type:** API (event-triggered)
- **Gateway:** Apollo CN
- **Security:** ADFS Clasp for integration authorization
- **Performance:** Per API < 1.5 sec
- **Trigger:** QR Code scan event

## Business Flow Impact

| Domain | Function | Type | Request | Response |
|--------|----------|------|---------|----------|
| Sales Management | TX Sync (Query Sales Order) | API | E-FAPIAO | SMART POS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Target System | SMART POS only provides complete order information |
| Target System | Invoice issuance logic fully controlled by E-FAPIAO |
| Target System | Tax rate managed by E-FAPIAO |
| Target System | Only positive sales get invoices (not returns) |
| Target System | For exchanges: same product → no invoice; different product → invoice for positive item only |
| Integration Gateway | Apollo CN required for SMART POS & E-FAPIAO |
| Security | ADFS Clasp for authorization |

## Low-Level Design

### Flow: TX Sync (Query Sales Order)
- **Trigger:** Event-based (QR Code scan)
- **Protocol:** API
- **Direction:** E-FAPIAO queries SMART POS for sales order data
- **Parallel Run:** TTPOS keeps AS-IS; SMART POS parallel run and cut-over use same design
