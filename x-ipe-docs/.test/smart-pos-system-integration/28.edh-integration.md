# [3 Divisions] SMART POS <-> EDH Integration Design

> Integration design for SMART POS and EDH (Enterprise Data Hub) for all three divisions, sending SOH, stock movement, sales target, and tender data for enterprise-level analytics.

## Integration Overview

EDH is the Enterprise Data Hub that receives multiple data feeds from SMART POS for analytics and reporting. All flows follow the same pattern: SMART POS → Blob Storage → Apollo → EDH.

### Key Facts
- **Divisions:** FBP (SOH, Stock Movement, SaleTarget), FSN/WFJ (Tender)
- **Integration Type:** File (all flows)
- **Gateway:** Apollo CN
- **Note:** SMART POS sends real-time SOH (not snapshot); uses TT Store code

## All Flows Between EDH & SMART POS

| Division | Flow | AS-IS | Parallel Run | Type | Source | Target |
|----------|------|-------|-------------|------|--------|--------|
| FBP | SOH | N/A | SMART POS → Blob → Apollo → EDH | File | SMART POS | EDH |
| FBP | Stock Movement | N/A | SMART POS → Blob → Apollo → EDH | File | SMART POS | EDH |
| FBP | SaleTarget | N/A | SMART POS → Blob → Apollo → EDH | File | SMART POS | EDH |
| FSN/WFJ | Tender | N/A | SMART POS → Blob → Apollo → EDH | File | SMART POS | EDH |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | SMART POS sends only SMART POS SOH/Movement to EDH |
| Enterprise Direction | SMART POS sends real-time SOH (not snapshot) |
