# [FSN] SMART POS <-> CASS Integration Design

> Integration design for SMART POS and CASS (Client After-Sales Service) for FSN division, covering sales transaction query, CASS key generation, and transaction fetching.

## Integration Overview

CASS provides client after-sales service capabilities. In the TO-BE architecture, CASS no longer integrates directly with POS — instead it works through **LION CRM** for CX-related TX data. The touchpoint for CASS binding has moved from CSE CN (retired) to **ICON**.

### Key Architectural Change
- **AS-IS:** CASS integrated directly with TTPOS; CSE CN was the touchpoint
- **TO-BE:** LION CRM is the golden source for CX-related Sales Transaction; ICON replaces CSE CN as the touchpoint

### Key Facts
- **Division:** FSN only
- **Integration Type:** API
- **Gateway:** Apollo CN
- **Security Note:** TX may contain PII — needs validation with security team

## Business Flow Impact

| Domain | Function | Type | Caller | Callee |
|--------|----------|------|--------|--------|
| Non-POS | Query Sales Transaction | API | ICON | SMART POS |
| Non-POS | Mark TX to Generate CASS Key | API | ICON | SMART POS |
| Non-POS | Fetch Sales TX by CASS Key | API | CASS | SMART POS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | After Sales is not part of SMART POS domain (focused on sales) |
| Enterprise Direction | LION contains both CX and TX — good channel for Non-POS domain |
| Target System | CASS business logic remains unchanged for SMART POS |
| Design Reference | Referenced from AS-IS TTPOS lessons learned |
| Integration Gateway | Apollo CN required for SMART POS integration |

## Architecture Comparison

### Query Sales Transaction & Generate CASS Key
- AS-IS: TTPOS was golden source for Sales Transaction
- TO-BE: LION CRM is golden source for CX-related Sales Transaction
- AS-IS: CSE CN was the touchpoint for query/CASS Key generation
- TO-BE: CSE CN retired → **ICON** replicates existing logic

### Query TTPOS by CASS Key
- AS-IS: CASS integrated with TTPOS to fetch sales TX by CASS Key
- TO-BE: CASS queries LION CRM with CASS Key for sales transaction

## Low-Level Design — Flow Deep Dives

1. **Query Sales Transaction** — ICON → SMART POS (API)
2. **Mark a Sales Transaction to Generate CASS Key** — ICON → SMART POS (API)
3. **Fetch Sales Transaction by CASS Key** — CASS → SMART POS (API)
