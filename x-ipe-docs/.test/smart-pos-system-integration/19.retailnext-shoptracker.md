# [FSN/WFJ/FBP] SMART POS <-> RetailNext & ShopTracker Integration Design

> Integration design for SMART POS and RetailNext/ShopTracker for all three divisions, syncing TX data for store traffic monitoring.

## Integration Overview

RetailNext and ShopTracker receive sales transaction data from SMART POS for store traffic monitoring and analysis. Uses the common TX distribution pattern via Apollo APAC.

### Key Facts
- **Divisions:** FSN, WFJ, FBP (all three)
- **Integration Type:** API
- **Gateways:** Apollo CN (SMART POS), Apollo APAC (TX distribution)
- **TX Canonical Version:** SMART POS v2.6.2 (vs TTPOS v2.2)

## Business Flow Impact

| Domain | Function | Downstream |
|--------|----------|------------|
| Sales Management | Sync Sales TX To RetailNext/ShopTracker | Monitor |

## Integration Table View

| Division | Flow | Type | Source | Target |
|----------|------|------|--------|--------|
| FBP | Sync TX | API | SMART POS | RetailNext/ShopTracker |
| FSN/WFJ | Sync TX | API | SMART POS | RetailNext/ShopTracker |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | Parallel run: TTPOS TX (v2.2) and SMART POS TX (v2.6.2) both sent |
| Enterprise Direction | SMART POS source system code differs from TTPOS |
| Enterprise Direction | Uses common TX distribution pattern via Apollo APAC |
| Target System | RetailNext/ShopTracker process flow assumed unchanged |
| Integration Gateway | Apollo CN for SMART POS; Apollo APAC for TX distribution |
