# [FSN/WFJ/FBP] SMART POS <-> RS Integration Design

> Integration design for SMART POS and RS (Replenishment System) for all three divisions, syncing stock on hand and sales transaction data for automated replenishment.

## Integration Overview

RS receives stock on hand (SOH) and sales transaction (TX) data from SMART POS to drive automatic replenishment order suggestions.

### Key Facts
- **Divisions:** FSN, WFJ, FBP (all three)
- **Integration Types:** File (SOH), API (TX)
- **Gateways:** Apollo CN (SMART POS), Apollo APAC (SOH & TX distribution)
- **TX Canonical Version:** SMART POS uses v2.6.2 (vs TTPOS v2.2)

## Business Flow Impact

| Domain | Function | Downstream |
|--------|----------|------------|
| Inventory Management | Sync Stock On Hand To RS | Auto Suggest Replenishment Order |
| Sales Management | Sync Sales Transaction To RS | Auto Suggest Replenishment Order |

## Integration Table View

| Division | Flow | Type | Source | Target |
|----------|------|------|--------|--------|
| FBP | Sync Merged SOH To RS | File | SMART POS | RS |
| FBP | Sync TX | API | SMART POS | RS |
| FSN/WFJ | Sync Merged SOH To RS | File | SMART POS | RS |
| FSN/WFJ | Sync TX | API | SMART POS | RS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | SMART POS two tenants — sends separate SOH per tenant |
| Enterprise Direction | Parallel run: SOH extracted from both POS, consolidated, sent to Apollo CN |
| Enterprise Direction | Parallel run: TTPOS TX (v2.2) and SMART POS TX (v2.6.2) both sent |
| Enterprise Direction | SMART POS TX source system code differs from TTPOS |
| Enterprise Direction | TX uses common pattern: forward to Apollo APAC for downstream distribution |
| Target System | RS process flow assumed unchanged |
| Integration Gateway | Apollo CN for SMART POS; Apollo APAC for distribution |

## Low-Level Design — Flow Deep Dives

1. **Sync Merged SOH To RS** — SMART POS → RS (File)
2. **Sync TX Data To RS** — SMART POS → RS (API via Apollo APAC distribution)
