# [FSN/WFJ/FBP] SMART POS <-> dataLAB Integration Design

> Integration design for SMART POS and dataLAB for all three divisions, syncing TX data, SOH, stock movement, deposit, credit note, and item reserved data for reporting.

## Integration Overview

dataLAB is a data analytics/reporting platform that receives multiple data feeds from SMART POS. In parallel run, SMART POS combines TTPOS and SMART POS data for some flows. Also integrates with Craft and GDP.

### Key Facts
- **Divisions:** All three (FSN, WFJ, FBP)
- **Integration Types:** API (TX), File (SOH, stock movement, etc.)
- **Gateways:** Apollo CN, Apollo APAC
- **Related systems:** Also integrates with Craft and GDP

## Business Flow Impact

| Domain | Function | Downstream |
|--------|----------|------------|
| Sales Management | Sync TX Data | Report |
| Inventory | Sync SOH Data | Report |
| Inventory | Sync Movement Data | Report |
| Sales | Sync Deposit Data | Report |
| Sales | Sync Credit Note Data | Report |
| Sales | Sync Item Reserved Data | Report |

## All Flows Between dataLAB & SMART POS

| Division | Flow | AS-IS | Parallel Run | To-Be | Type |
|----------|------|-------|-------------|-------|------|
| 3 Div | TX Data | TTPOS → Apollo CN → Apollo APAC → dataLAB | Both TTPOS & SMART POS send (same format, same source code) | SMART POS → Apollo CN → Apollo APAC → dataLAB | API |
| 3 Div | SOH | TTPOS → Apollo CN(ADF) → dataLAB Blob → dataLAB | SMART POS combines TTPOS + SMART POS data | SMART POS → Blob → Apollo CN → dataLAB Blob → dataLAB | File |
| 3 Div | Stock Movement | Same as SOH | SMART POS combines both | Same as SOH | File |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | Focus on parallel run phase |
| Enterprise Direction | TX: both TTPOS and SMART POS send separately |
| Enterprise Direction | SOH/Movement/Item Reserved/Deposit/Credit Note: SMART POS combines both systems' data |
| Enterprise Direction | Staff Master/TMO Code/Campaign: Keep AS-IS from TTPOS; ops team ensures double maintenance |
| Integration Gateway | Apollo CN/APAC required |

## Low-Level Design — Flow Deep Dives

1. **Sync TX Data To dataLAB** — SMART POS → Apollo CN → Apollo APAC → dataLAB (API)
2. **Sync SOH To dataLAB** — SMART POS → Blob → Apollo CN → dataLAB Blob → dataLAB (File)
