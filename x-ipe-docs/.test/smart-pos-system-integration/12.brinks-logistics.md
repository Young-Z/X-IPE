# [WFJ] SMART POS <-> Brinks Logistics Integration Design

> Integration design for SMART POS and Brinks Logistics for WFJ division, handling logistics-related transaction data sync for high-value WFJ items.

## Integration Overview

Brinks Logistics queries SMART POS for sales order data to manage logistics for WFJ (Watches & Fine Jewelry) high-value items.

### Key Facts
- **Division:** WFJ only
- **Integration Type:** API
- **Gateway:** Apollo CN
- **Security:** Basic Authorization (proposed by Brinks based on system adjustment difficulty and timeline)
- **Performance:** Per API < 1.5 sec
- **Domain:** Inventory Management

## Business Flow Impact

| Domain | Function | Type | Request | Response |
|--------|----------|------|---------|----------|
| Inventory Management | TX Sync (Query Sales Order) | API | Brinks | SMART POS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Integration Gateway | Apollo CN required for Brinks & SMART POS |
| Security | Basic Authorization (Brinks requirement based on system limitations and timeline) |

## Low-Level Design

### Flow: TX Sync — Query Sales Order
- **Protocol:** API
- **Direction:** Brinks queries SMART POS
- **Parallel run and cut-over:** Same design
