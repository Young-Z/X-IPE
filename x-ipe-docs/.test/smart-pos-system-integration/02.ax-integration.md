# [FSN/WFJ/FBP] SMART POS <-> AX Integration Design

> Integration design for SMART POS and Microsoft Dynamics AX (ERP system) covering inventory, sales, and product data flows across all three divisions (FSN, WFJ, FBP).

## Integration Overview

AX (Microsoft Dynamics AX) is the core ERP system integrated with SMART POS for inventory management, sales transactions, and product/price master data synchronization. This integration covers all three divisions (FSN, WFJ, FBP) and is mediated through **Apollo APAC** and **Apollo CN** as integration gateways.

### Key Architectural Decisions
- Uses **TTPOS boutique code** as the only boutique code between AX and SMART POS (no conversion required by Apollo)
- **Apollo CN** is required to integrate with SMART POS
- Only **Apollo APAC** is allowed to push to AX and TTPOS file server
- **PII Data** (e.g., VIP Code) must be removed before sending to AX to meet CBDT Policy
- Integration method: **File-based** via Azure Blob Storage through Apollo

### Architecture Comparison (AS-IS vs TO-BE)
- AS-IS: AX and TTPOS directly integrated without middleware
- TO-BE: AX integrates via Apollo APAC/CN with SMART POS; all TTPOS flows migrated

## Flow-Level Deep Dives (Documented)

1. **DC Transfer to Store (TI)** — AX → SMART POS via Apollo
2. **DC Transfer to Store ACK (TIAC)** — SMART POS → AX via Apollo
3. **Store Return to DC (TO)** — SMART POS → AX via Apollo
4. **Store Return to DC ACK (TOAC)** — AX → SMART POS via Apollo
5. **Store Stock Adjustment (IM)** — SMART POS → AX via Apollo
6. **FBP Exists Replenishment (E01, E02)** — SMART POS → AX via Apollo
7. **Store To Store Transfer (TO, TOAC - FSN/WFJ)** — SMART POS → AX via Apollo
8. **Store To Store Transfer (TO, TOAC, TOCX - FBP)** — SMART POS → AX via Apollo
9. **Transaction** — SMART POS → AX via Apollo
10. **Product Master** — AX → SMART POS via Apollo
11. **Price Master** — AX → SMART POS via Apollo
