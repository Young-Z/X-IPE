# [FSN/WFJ/FBP] SMART POS <-> WMS Integration Design

> Integration design for SMART POS and WMS (Warehouse Management System) for all three divisions, syncing shipment orders for store receiving.

## Integration Overview

WMS sends shipment order files to SMART POS for store receiving operations. WMS continues to send all files to AX; only shipment orders go to SMART POS.

### Key Facts
- **Divisions:** All three (FSN, WFJ, FBP)
- **Integration Type:** File
- **Gateway:** Apollo CN
- **Performance Target:** 400 files/5min (target), 3000 files/hour
- **Domain:** Inventory

## Business Flow Impact

| Domain | Function | Type | Source | Target | Downstream |
|--------|----------|------|--------|--------|------------|
| Inventory | Sync Shipment Order | File | WMS | SMART POS | Receiving |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | WMS sends all files to AX; only shipment orders to SMART POS |
| Enterprise Direction | No change to WMS system — just modify copy & move BAT file logic |
| Enterprise Direction | Apollo doesn't filter — just moves files; SMART POS handles 3 division data |
| Target System | WMS process flow assumed unchanged |
| Integration Gateway | Apollo CN required |
