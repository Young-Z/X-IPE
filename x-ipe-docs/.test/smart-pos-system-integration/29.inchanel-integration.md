# [FSN] SMART POS <-> InChanel Integration Design

> Integration design for SMART POS and InChanel for FSN division, providing SOH query and stock change push services for omni-channel stock visibility.

## Integration Overview

InChanel is an omni-channel system for FSN that queries stock on hand (SOH) from SMART POS and receives stock change notifications. InChanel and ICON are assumed to share the same SOH logic.

### Key Facts
- **Division:** FSN only
- **Integration Type:** API (bidirectional)
- **Gateways:** Apollo CN + Global APIM (FSN service catalog)
- **Auth:** ADFS Clasp

## Integration Table View

| Division | Flow | Type | Request System | Response System |
|----------|------|------|----------------|-----------------|
| FSN | FSN SOH Query | API | InChanel | SMART POS |
| FSN | Change Stock Push | API | SMART POS | FSN Service Catalog |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Target System | InChanel & ICON share the same SOH logic |
| Integration Gateway | Global APIM required for FSN service catalog |
| Integration Gateway | Apollo CN required for SMART POS & InChanel |
| Security | ADFS Clasp for authorization |
