# [FBP] SMART POS <-> FBRS Integration Design

> Integration design for SMART POS and FBRS (Fashion Business Replenishment System) for FBP division, managing suggested replenishment orders.

## Integration Overview

FBRS handles automated store replenishment for FBP division. FBRS generates suggested replenishment orders which are sent to SMART POS for confirmation, and confirmed orders are synced back.

### Key Facts
- **Division:** FBP only
- **Integration Type:** File (bidirectional)
- **Gateway:** Apollo CN & Apollo APAC
- **Domain:** Inventory

## Business Flow Impact

| Domain | Function | Type | Source | Target |
|--------|----------|------|--------|--------|
| Inventory | Sync Suggest Replenishment Order | File | FBRS | SMART POS |
| Inventory | Sync Replenishment Order | File | SMART POS | FBRS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | During parallel run, FBRS maintains store list of SMART POS stores |
| Enterprise Direction | Stores still on TTPOS keep AS-IS email flow |
| Enterprise Direction | Stores switched to SMART POS receive CSV via file integration |
| Integration Gateway | Apollo CN & Apollo APAC required |
| Security | Defined at flow level |

## Low-Level Design — Flow Deep Dives

1. **Sync Suggest Replenishment Order** — FBRS → SMART POS (File/CSV)
2. **Sync Replenishment Order** — SMART POS → FBRS (File)
