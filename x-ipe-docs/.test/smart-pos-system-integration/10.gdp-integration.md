# [BCP][FBP] SMART POS <-> GDP Integration Design

> Integration design for SMART POS and GDP (Global Distribution Platform) for BCP/FBP divisions, handling stock on hand data synchronization.

## Integration Overview

GDP receives stock on hand (SOH) data from SMART POS for inventory management across FBP and FSN/WFJ divisions.

### Key Facts
- **Divisions:** BCP, FBP
- **Integration Type:** Data sync (SOH)
- **Gateway:** Apollo CN (integration), Apollo APAC (SOH distribution)
- **Domain:** Inventory Management

## Business Flow Impact

| Domain | Function | Type | Source | Target |
|--------|----------|------|--------|--------|
| Inventory Management | Sync Stock On Hand To GDP | Data Sync | SMART POS | GDP |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | SMART POS has two tenants (FBP and FSN/WFJ) — sends separate SOH data per tenant |
| Enterprise Direction | During parallel run, SOH extracted from both TTPOS and SMART POS, consolidated, then sent to GDP |
| Target System | GDP process flow assumed unchanged for SMART POS |
| Integration Gateway | Apollo CN required for SMART POS integration |
| Integration Gateway | Apollo APAC required for SOH distribution pattern |
| Security | Defined at flow level |
