# [FSN/WFJ/FBP] SMART POS <-> Workday Integration Design

> Integration design for SMART POS and Workday (via Employee Platform) for all three divisions, syncing employee master data for staff management at stores.

## Integration Overview

Workday (Employee Platform) sends employee master data to SMART POS daily as JSON files. Two separate files are generated — one for FBP and one for FSN/WFJ — routed to different storage folders.

### Key Facts
- **Divisions:** All three (FSN, WFJ, FBP)
- **Integration Type:** File (JSON)
- **Gateway:** Apollo CN
- **Frequency:** Daily
- **Source:** Employee Platform (Workday)

## Integration Table View

| Division | Flow | Type | Source | Target |
|----------|------|------|--------|--------|
| 3 Div | Employee Master | File | Employee Platform | SMART POS |

## Integration Findings & Assumptions

| Category | Finding/Assumption |
|----------|-------------------|
| Enterprise Direction | Workday sends employee JSON to POS daily |
| Enterprise Direction | Two files generated daily: one for FBP, one for FSN/WFJ |
| Enterprise Direction | Apollo moves files from Workday to SMART POS Storage (two files → two folders) |
| Integration Gateway | Apollo CN required |

- **Sync Employee Master from Employee Platform to SMART POS** — Daily file-based sync
