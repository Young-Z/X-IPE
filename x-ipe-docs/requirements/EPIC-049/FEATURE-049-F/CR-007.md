## Description

Two improvements to the Normal Upload zone in the KB browse modal:

1. **Upload success indication:** After files are uploaded via the Normal Upload zone (drag-and-drop or browse), show a visible success message indicating how many files were uploaded successfully. Currently, uploads complete silently with no visual feedback.

2. **Click-to-browse file selection:** The Normal Upload zone currently only supports drag-and-drop for direct upload within the browse modal. Clicking the zone opens the full `KBFileUpload` modal (closing the browse modal). Instead, clicking the zone (or the "browse" text) should open a native file selection dialog, matching the behavior of the Intake zone which already has this via `_triggerIntakeFileInput()`.

## Business Justification

- **Success indication:** Users have no way to know if their upload succeeded without checking the file list. Standard UX practice requires upload feedback.
- **Click-to-browse:** Users expect to click the upload zone to select files — the "browse" text is already displayed but clicking opens a different modal instead. The Intake zone already supports click-to-browse, creating an inconsistency.

## Proposed Approach

1. **Click-to-browse:** Replace the `trigger-upload` action handler (which currently closes browse modal and opens `KBFileUpload`) with a native file input approach matching `_triggerIntakeFileInput()`. Create `_triggerNormalFileInput()` that creates an `<input type="file" multiple>`, triggers click, and uploads selected files via `_uploadFiles()`.

2. **Success indication:** After `_uploadFiles()` completes successfully, show a temporary success message in the upload zone (e.g., "✅ {N} file(s) uploaded") that auto-clears after a few seconds.
