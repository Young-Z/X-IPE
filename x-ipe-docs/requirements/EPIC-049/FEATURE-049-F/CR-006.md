## Description

The document and image rendering area in the KB article view (browse scene) should span the full available width with appropriate left/right margins, instead of being constrained to `max-width: 780px`. Currently, the `.kb-article-main` container has a hardcoded `max-width: 780px` which wastes significant horizontal space, especially when viewing images, PDFs, and other documents.

The selected element from the feedback is `div.kb-article-header-block` inside the article layout. The change affects the parent container `.kb-article-main` which constrains the overall content width.

## Business Justification

- Images and documents appear unnecessarily small on modern wide screens
- Significant white space is wasted on both sides of the content area
- Industry standard (Notion, Confluence, GitHub) uses wider content areas for media preview
- Better utilization of screen real estate improves user experience when browsing KB content

2. Add appropriate left/right padding (e.g., 32px–48px) to maintain readability margins
3. Ensure images, iframes (PDF/docx/HTML), and code blocks scale properly within the wider container
4. Verify sidebar positioning remains correct at all viewport widths
