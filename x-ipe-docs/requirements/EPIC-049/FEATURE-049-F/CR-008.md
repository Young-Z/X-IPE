## Proposed Approach

1. **Extract** a shared `FilePreviewRenderer` class from deliverable-viewer.js's `showPreview()` pattern into `src/x_ipe/static/js/core/file-preview-renderer.js`
2. **API:** `renderPreview(filePath, container, options)` where options includes `{ apiEndpoint, onError }`
3. **File type detection:** Internal, based on file extension (mirrors deliverable-viewer.js logic)
4. **Supported formats:** Markdown+DSL (via ContentRenderer), Images, DOCX/MSG (via X-Converted detection), PDF, HTML (sandboxed iframe), Code/Text
5. **KB browse modal** uses FilePreviewRenderer instead of direct rendering
6. **Deliverable viewer** refactored to use FilePreviewRenderer (eliminating duplication)
7. **Configurable endpoint:** Supports both `/api/kb/files/{path}/preview` and `/api/ideas/file?path=` via options
