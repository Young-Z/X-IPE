## Description

Fundamental redesign of the EPIC-054 Behavior Tracker:
1. **Simplify tracker-toolbar.js to <5KB** — minimal HTML, no elaborate styles. Simple buttons: Start, Stop, Analysis
2. **Move control to skill script** — 5-second polling by `track_behavior.py` via Chrome DevTools MCP `evaluate_script`. No LLM processing until user clicks "Analysis" button
3. **Screenshot on tracking list change** — skill script detects new events via polling diff → takes screenshot via Chrome DevTools MCP `take_screenshot`
4. **URL change reinjection** — skill script detects URL change during polling → reinjects tracker → loads data from localStorage
5. **New output folder structure** — `x-ipe-docs/learning/{semantic-name}/track/track-list.json` for events, `x-ipe-docs/learning/{semantic-name}/imgs/` for screenshots
6. **Analysis button** — IIFE sets flag `window.__xipe_analysis_requested = true`; skill script detects flag during poll → triggers LLM post-processing; writes corrected events back

## Business Justification

The current 573-line IIFE (23KB) with Shadow DOM, glass-morphism, virtual scrolling, and elaborate PII UI is overengineered for AI agent training data collection. The actual consumer is a developer, not an end-user. Moving control from the IIFE to the Python skill script via Chrome DevTools MCP:
- Reduces injected code complexity (from 23KB to <5KB)
- Leverages Chrome DevTools MCP capabilities (screenshots, URL detection) instead of reimplementing in JS
- Enables persistent storage via project folder structure instead of ephemeral JSON files
- Separates concerns: JS captures events passively, Python orchestrates actively

## Proposed Approach

### Architecture Inversion

**Before (IIFE-centric):** JS IIFE handles everything — recording, UI, backup, lifecycle. Python just injects and collects at end.

**After (Script-centric):** JS IIFE is a passive event capture buffer with minimal UI (3 buttons). Python script polls every 5s, manages screenshots, detects URL changes, handles reinjection, manages folder output.

### New tracker-toolbar.js (<5KB)

Keep:
- PIIMasker (simplified — no whitelist UI)
- CircularBuffer (unchanged)
- EventSerializer (simplified — fewer metadata fields)
- RecordingEngine (unchanged core — capture-phase listeners)

Remove:
- TrackerToolbox Shadow DOM with glass-morphism (replace with minimal bar)
- BackupManager (simplified to just localStorage get/set)
- Event list display, PII tag UI, drag handlers, virtual scrolling

Add:
- Simple fixed-bottom bar: `[● Start] [■ Stop] [📊 Analysis]` + event counter
- `window.__xipe_analysis_requested` flag for Analysis button
- `window.__xipe_status` for script to read current state

### New track_behavior.py

Add:
- `poll_loop(interval=5)` — calls evaluate_script to collect events, diff against previous
- `take_screenshot_on_change()` — if new events detected, call MCP take_screenshot
- `detect_url_change()` — compare current URL with last known; reinject if changed
- `manage_output_folder(name)` — create `x-ipe-docs/learning/{name}/track/` and `imgs/`
- `write_track_list(events)` — write to track-list.json (append mode)
- `check_analysis_flag()` — check `window.__xipe_analysis_requested`; if true, trigger post-processing
