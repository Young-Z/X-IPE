## Description

Add a new operation `start_active_tracking` to `x-ipe-knowledge-mimic-web-behavior-tracker` that bundles three capabilities present in the deprecated `x-ipe-tool-learning-behavior-tracker-for-web` but missing from the current mimic skill:

1. **5-second polling loop** — periodically calls the in-page tracker to drain its event buffer into the session directory
2. **Auto-reinject on URL change** — detects navigation away from the originally injected page and re-injects the tracker IIFE (clearing the `__xipeBehaviorTrackerInjected` guard first)
3. **Auto-screenshot on new events** — when the polling tick detects new events since the last tick, captures a screenshot via Chrome DevTools MCP and stores it under the session directory

The existing `start_tracking` operation (stateless, single-shot injection) is preserved unchanged. The new `start_active_tracking` is an opt-in, stateful variant for sessions that need continuous capture without requiring the orchestrator to drive the loop.

## Business Justification

Without these three capabilities, the new mimic skill is functionally narrower than the deprecated tool skill it replaces. We cannot retire `x-ipe-tool-learning-behavior-tracker-for-web` while a working capability gap exists, leaving two parallel skills with overlapping (but non-equivalent) function. Adding this op closes the gap and lets us cleanly remove the deprecated skill in a follow-up.

The polling loop is also necessary for long sessions where the in-page circular buffer (default 10,000 events) would otherwise silently prune older events.

- **Execution model:** Agent-driven loop, NOT detached subprocess — matches deprecated skill exactly. The skill exposes `inject` / `poll_tick` / `stop` building blocks; the orchestrating agent runs the 5s timing.
- **File layout:** Single accumulating `track-list.json` (overwritten each tick with full event list, schema_version 2.0) — matches old skill's `write_track_list` behavior. NO per-tick JSON files.
