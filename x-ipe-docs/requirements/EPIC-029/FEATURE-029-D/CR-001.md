## Description

Add a small expand/collapse toggle directly on the session explorer's resize border (drag handle). Currently, the only way to toggle the explorer panel is via the header toggle button. The resize handle should also support click-to-toggle: if the user clicks the border without dragging, the panel should expand or collapse. When collapsed, the border should remain visible as a thin clickable strip instead of being completely hidden.

## Business Justification

Border-based toggles are a standard UX pattern in modern IDEs and panel-based applications (VS Code sidebar, IntelliJ tool windows). This improves discoverability — users intuitively expect that clicking the border between two panels will toggle visibility. The current header toggle button is small and less discoverable.

## Proposed Approach

1. **Click vs Drag Detection:** On mousedown, record start position. On mouseup, if total movement < 3px threshold, treat as click → call `toggleExplorer()`. If movement ≥ threshold, treat as drag (existing behavior).
2. **Visible Handle When Collapsed:** Instead of `display: none`, keep the resize handle visible when collapsed. Style it as a thin strip (~5px) with a chevron indicator (`›` when collapsed, `‹` when expanded).
3. **CSS Transition:** The chevron rotates/changes on toggle with the existing 0.25s transition.
4. **Reuse `toggleExplorer()`:** Click-to-toggle calls the same `toggleExplorer()` method, ensuring all state persistence and re-fit logic is preserved.
