## Description

Avoid exposing "DAO" (道) terminology in external-facing skill descriptions and inter-skill interfaces. The concept of 道 is culturally specific to Chinese philosophy and may be misunderstood or misinterpreted by non-Chinese-culture users when used directly as an API/interface term.

**Changes required:**
1. External-facing frontmatter `description` in DAO skills should use universally understood language (e.g., "human proxy mediation", "end-user guidance")
2. Inter-skill documentation and template descriptions should avoid "DAO" as the primary descriptor
3. 道 should be positioned as the **internal CORE backbone** within DAO skills — the philosophical reasoning framework — not the external-facing brand
4. Skill type name (`x-ipe-dao`) and directory names (`x-ipe-assistant-user-representative-Engineer`) remain unchanged per user confirmation

**Additional fixes bundled:**
- Fix AC-047-A.6: DAO template missing embedded 7-step backbone (currently has generic placeholders)
- Align technical design input/output contract with actual implementation

## Business Justification

DAO/道 is an abstract concept rooted in Chinese cultural philosophy. Without that context, the term can be confusing or misinterpreted. Using universally understood language in external-facing surfaces improves accessibility and reduces onboarding friction for all contributors, while preserving the philosophical depth internally.
