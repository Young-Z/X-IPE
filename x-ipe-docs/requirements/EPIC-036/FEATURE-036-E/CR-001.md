## Description

The deliverables section in the workflow panel currently groups all deliverables by **stage** (Ideation Deliverables, Requirement Deliverables, Implement Deliverables, Validation Deliverables). Per-feature deliverables from the implement and validation stages are nested under feature labels within their respective stage sections.

The change request is to reorganize the layout so that **each feature has its own section** listing all its deliverables, rather than having per-feature deliverables scattered across multiple stage sections. Shared/global deliverables (from ideation and requirement stages) remain in their own section.

**Current layout:**
```
Ideation Deliverables
  [shared cards]
Requirement Deliverables
  [shared cards]
Implement Deliverables
  Feature A: [cards]
  Feature B: [cards]
Validation Deliverables
  Feature A: [cards]
  Feature B: [cards]
```

**Proposed layout:**
```
Shared Deliverables
  [ideation + requirement cards, grouped by stage]
Feature A
  [all Feature A deliverables across stages]
Feature B
  [all Feature B deliverables across stages]
```

## Business Justification

When a workflow has multiple features, users need to quickly see all deliverables for a specific feature in one place. The current stage-grouped layout forces users to scan across multiple sections to find all deliverables for one feature. Feature-grouped layout better matches the mental model of "what did Feature X produce?"

## Proposed Approach

1. Modify `_renderDeliverables()` in `workflow-stage.js`:
   - Keep shared deliverables (ideation + requirement stages) in a "Shared Deliverables" section
   - Group per-feature deliverables by `feature_id` into feature sections
   - Each feature section shows all its deliverables (across implement, validation, feedback stages)
2. Update CSS in `workflow.css`:
   - Add `.deliverables-feature-section` styling
   - Reuse existing card rendering (DeliverableViewer, renderCard)
3. No backend changes needed
