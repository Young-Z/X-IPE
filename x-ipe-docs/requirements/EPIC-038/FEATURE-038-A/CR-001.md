## Description

When the workflow auto-proceed dropdown is set to "auto" or "stop_for_question", the CLI command generated by the Action Execution Modal should append an `--execution@{mode}` flag:
- `auto` → `--execution@{keep running}`
- `stop_for_question` → `--execution@{keep running stop only on question}`
- `manual` → no flag appended

## Business Justification

The auto-proceed dropdown setting needs to be communicated to the CLI agent so it knows whether to auto-proceed through workflow steps or stop for human input. Without this flag, the agent defaults to manual behavior regardless of the dropdown setting.

## Proposed Approach

1. Add `_autoProceed` property to constructor (default: 'manual')
2. Add `_loadAutoProceed()` method that reads auto_proceed from cached workflow instance
3. Add `_buildExecutionFlag()` helper that maps mode to CLI flag
4. Update `_buildCommand()` to append execution flag before `--workflow-mode`
5. Update `_handleExecute()` workflow path to also append execution flag
