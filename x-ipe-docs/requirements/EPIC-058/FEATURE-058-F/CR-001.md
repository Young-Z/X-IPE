## Description

Add a WebSocket callback channel from the AI Agent's ontology search to the KB Graph Viewer UI. When the user clicks "Search with AI Agent" and the AI agent executes a search via `search.py`, the search results (matched nodes + related subgraph) should be pushed back to the frontend via a SocketIO event. The frontend then renders the subgraph in the Cytoscape.js canvas in real-time — highlighting direct matches and BFS-expanded neighbors.

**Current behavior:** AI Agent search results are displayed only in the terminal console. The graph viewer canvas is unaware of agent-driven searches.

**Desired behavior:** AI Agent search results are simultaneously displayed in the terminal AND pushed to the graph viewer canvas via WebSocket, providing a live visual representation of what the agent found.

## Business Justification

The current "Search with AI Agent" flow has a gap: the AI agent finds results in the terminal, but the graph viewer stays static. Users must manually re-search in the graph UI to see results visually. This creates a disconnected experience. The WebSocket callback closes the loop — agent searches trigger real-time graph updates, providing immediate visual context for the AI's findings.

## Proposed Approach

1. **Backend: Socket handler** — Create `src/x_ipe/handlers/ontology_handlers.py` with a `register_ontology_handlers(socketio)` function. Register in `app.py`.
2. **Backend: Callback bridge** — Create `.github/skills/x-ipe-tool-ontology/scripts/ui-callback.py` that calls the Flask server's internal socket emit (via HTTP POST to a new internal endpoint, or direct `socketio.emit()`).
3. **Frontend: Socket listener** — In `ontology-graph-viewer.js`, when "Search with AI Agent" is clicked, subscribe to `ontology_search_result` socket events. On receiving data, call `canvas.highlightSubgraph()` and optionally render the dropdown.
4. **Data flow**: `search.py` output → `ui-callback.py` transforms to Cytoscape format → emits via socket → frontend renders.
