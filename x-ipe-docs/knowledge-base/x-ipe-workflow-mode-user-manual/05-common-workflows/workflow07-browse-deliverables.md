# Browse Workflow Deliverables

## Overview
Every workflow action produces deliverables — files and folders that are tracked and viewable from the workflow card.

## Step-by-Step

### Step 1: Find Deliverables Section
1. Expand a workflow card
2. Scroll down past Feature Lanes to the **Deliverables** section
3. A count badge shows total deliverables (e.g., "52 ▾")

### Step 2: Understand the Organization

Deliverables are organized in two tiers:

| Section | Content |
|---------|---------|
| **SHARED DELIVERABLES** | Files from Ideation and Requirement stages (shared across all features) |
| **PER-FEATURE** | Files grouped by feature ID (e.g., "FEATURE-050-A — Extractor Skill Foundation") |

### Step 3: Identify File Types

| Icon | Type | Example |
|------|------|---------|
| 💡 | Idea files | `new idea.md`, `idea-summary-v1.md` |
| 📋 | Requirement docs | `specification.md`, `technical-design.md` |
| 💻 | Code files | `SKILL.md`, source files |
| 📊 | Test reports | `acceptance-test-report.md` |
| 📁 | Folders | Expandable directory entries |

### Step 4: View a File
1. Click on a **file** entry → opens file preview (in-app viewer)
2. Each file shows its relative path below the name

### Step 5: Browse a Folder
1. Click on a **�� folder** entry → opens Folder Browser Modal
2. See [feature06-deliverables-browser](../04-core-features/feature06-deliverables-browser.md) for full details
3. Left panel: file tree | Right panel: file preview
4. Search, navigate, and download files
